# DisasterAgent：自然灾害领域多智能体系统

基于 **LangGraph** 构建的自然灾害领域多智能体问答系统。该系统将用户的复杂查询分解为结构化的子任务计划，动态地将每个子任务路由到最合适的领域专家智能体，并综合生成最终的完整回答。

### 核心工作流程

1. **规划阶段**：规划器 LLM 分析用户问题，识别核心和补充信息需求，生成结构化的 JSON 计划，指定每个子任务应咨询的专家。

2. **调度与执行阶段**：调度器逐步将每个子任务分派给指定的专家智能体。收到响应后，评估是否已收集到足够的信息，或是否需要进一步咨询其他专家。

3. **回答综合阶段**：当所有子任务完成（或达到最大交互次数限制）后，综合器汇总收集到的证据，生成连贯的最终回答。

## 专家智能体

| 智能体 | 能力 | 后端实现 |
|--------|------|----------|
| **法律法规智能体** | 中国灾害相关法律、法规、技术标准和应急预案 | RAG 管线，采用混合检索（通过 Milvus 实现稠密 + 稀疏向量检索）、重排序、相关性评分和幻觉检测 |
| **领域知识智能体** | 自然灾害成因、特征和分类的理论知识 | 基于 LLM 的问答 |
| **灾害记录智能体** | 结构化历史灾害数据查询（中国） | 通过 MCP 工具服务器实现 NL2SQL，覆盖地震记录、灾害损失、区域统计等 |
| **应急科普智能体** | 防灾减灾、应急响应指导和公共安全建议 | 基于 LLM 的问答 |
| **网络搜索智能体** | 实时网络信息检索，获取最新新闻、全球灾害数据和补充信息 | 通过 MCP 集成实现网络搜索 |

## 关键技术组件

- **LangGraph**：用于有状态、基于图的多智能体编排，支持条件路由
- **混合检索（RAG）**：由 **Milvus** 驱动的稠密（HNSW）+ 稀疏（BM25 风格）向量搜索，使用 **BGE-M3** 嵌入模型和 **BGE-Reranker-v2** 进行结果重排序
- **NL2SQL**：通过 MCP 工具服务器实现自然语言到 SQL 的转换，用于结构化灾害数据库查询
- **MCP（模型上下文协议）**：标准化的工具集成，用于网络搜索和数据库访问
- **多级质量控制**：RAG 管线内的文档相关性评分、幻觉检测和答案-问题对齐检查

## 项目结构

```
Plan_Module/
├── src/
│   ├── agent/              # 主图：规划、调度和综合
│   ├── LawRag_Subgraph/    # 法律法规 RAG 智能体（检索 + 重排序 + 生成）
│   ├── Knowledge_Subgraph/ # 领域知识智能体
│   ├── Document_Subgraph/  # 灾害记录智能体（NL2SQL）
│   ├── Emergency_Subgraph/ # 应急科普智能体
│   ├── WebSearch_Subgraph/ # 网络搜索智能体
│   ├── Asyn_hybridSearch/  # 异步混合向量搜索引擎
│   ├── qwen_model/         # LLM 封装和结构化输出工具
│   └── zerollm/            # RAG 管线组件（评分器、路由器、生成器等）
├── prompt/                 # 规划、路由和综合的提示词模板
├── tests/                  # 单元测试和集成测试
├── langgraph.json          # LangGraph 部署配置
└── pyproject.toml          # 项目依赖和元数据
```

## 技术栈

- **大语言模型**：Qwen2.5 系列（通过 vLLM / OpenAI 兼容 API 提供服务）
- **向量数据库**：Milvus
- **嵌入模型**：BGE-M3
- **重排序模型**：BGE-Reranker-v2-M3
- **框架**：LangGraph + LangChain
- **工具协议**：MCP（模型上下文协议）
